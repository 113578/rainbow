# Rainbow: Combining Improvements in Deep Reinforcement Learning

Реализация и экспериментальное исследование по статье:

> Hessel, M., Modayil, J., van Hasselt, H., Schaul, T., Ostrovski, G., Dabney, W., Horgan, D., Piot, B., Azar, M., & Silver, D. (2018). *Rainbow: Combining Improvements in Deep Reinforcement Learning*. AAAI 2018. [arXiv:1710.02298](https://arxiv.org/abs/1710.02298)

## Алгоритм

Rainbow DQN объединяет шесть расширений алгоритма DQN:

1. **Double Q-learning** — разделяет выбор и оценку действия для уменьшения переоценки
2. **Приоритизированное воспроизведение опыта** — выбирает переходы пропорционально величине TD-ошибки
3. **Дуэльная архитектура сети** — раздельно оценивает ценность состояния и преимущество действий
4. **Многошаговое обучение** (n=3) — использует усечённые n-шаговые награды для быстрого распространения наград
5. **Распределённое RL** (C51) — обучает категориальное распределение наград вместо среднего
6. **Шумовые сети** — заменяет epsilon-greedy исследование параметрическим шумом в весах сети

Реализация адаптирована из [Kaixhin/Rainbow](https://github.com/Kaixhin/Rainbow), модернизирована для Python 3.13 и Gymnasium.

## Выбранные эксперименты

Из полного набора экспериментов статьи выбраны 5, наилучшим образом демонстрирующих возможности алгоритма в рамках практических вычислительных ограничений.

### Обоснование выбора экспериментов

Статья проводит два типа экспериментов: (1) сравнение полного Rainbow с базовыми алгоритмами на 57 играх Atari, и (2) абляционные исследования с удалением по одному компоненту. С учётом вычислительных ограничений выбраны:

**Эксперимент 1: Полный Rainbow на Pong**
- *Почему:* Pong — простейшая игра Atari с быстрой сходимостью, идеальная для валидации реализации. Агент должен быстро освоить почти оптимальную стратегию, обеспечивая надёжную базовую линию.

**Эксперимент 2: Полный Rainbow на Breakout**
- *Почему:* Breakout — более сложная игра, требующая долгосрочной стратегии (пробить туннель через кирпичи). Тестирование на второй игре проверяет обобщаемость реализации.

**Эксперимент 3: Абляция — без приоритизированного воспроизведения**
- *Почему:* Статья определяет приоритизированное воспроизведение как один из двух наиболее важных компонентов (Рисунок 3), с его удалением производительность падает в 53 из 57 игр. Эта абляция проверяет воспроизводимость данного результата.

**Эксперимент 4: Абляция — без многошагового обучения (n=1)**
- *Почему:* Многошаговое обучение — второй по важности компонент согласно абляционному исследованию. Его удаление ухудшает как скорость раннего обучения, так и итоговую производительность. Использование n=1 (стандартные 1-шаговые награды) изолирует этот эффект.

**Эксперимент 5: Абляция — без распределённого RL**
- *Почему:* Распределённое Q-обучение занимает третье место по важности. Интересно, что в статье различий в раннем обучении не наблюдается — разрыв появляется только после ~40M кадров. Даже в коротком эксперименте эта абляция проверяет влияние распределения на динамику обучения.

### Что мы не включаем и почему

- **Полный бенчмарк на 57 играх:** Требует ~570 GPU-дней (10 дней × 57 игр). Нереализуемо.
- **Сравнения с базовыми алгоритмами (DQN, DDQN, A3C и др.):** Это хорошо известные результаты из литературы. Повторная реализация добавляет вычислительные затраты без научной ценности.
- **Абляции no-double и no-dueling:** Статья показывает ограниченное влияние на медианную производительность (Рисунки 3-4). Эффект double Q-learning маскируется диапазоном носителя распределения [-10, 10], а dueling даёт смешанные результаты по играм.
- **Абляция no-noisy:** Хотя шумовые сети помогают в среднем, эффект зависит от игры — в некоторых играх epsilon-greedy работает лучше. Менее однозначный результат, чем у выбранных абляций.

## Структура проекта

```
rainbow/
├── rainbow/
│   ├── __init__.py
│   ├── agent.py          # Агент Rainbow с поддержкой абляций
│   ├── env.py            # Обёртка среды Atari (Gymnasium)
│   ├── memory.py         # Приоритизированное воспроизведение (дерево отрезков)
│   ├── model.py          # NoisyLinear + дуэльная архитектура DQN
│   └── test.py           # Утилиты оценки
├── experiments/
│   ├── 01_rainbow_pong.sh          # Полный Rainbow на Pong
│   ├── 02_rainbow_breakout.sh      # Полный Rainbow на Breakout
│   ├── 03_no_priority_pong.sh      # Абляция: без приоритизации
│   ├── 04_no_multistep_pong.sh     # Абляция: без многошагового
│   ├── 05_no_distributional_pong.sh # Абляция: без распределения
│   ├── run_all.sh                  # Запуск всех экспериментов
│   ├── plot_results.py             # Построение графиков
│   └── results/                    # Результаты (создаётся автоматически)
├── main.py                # Точка входа (Typer CLI)
├── pyproject.toml         # Зависимости
└── README.md
```

## Использование

### Установка

```bash
uv sync
```

### Запуск всех экспериментов

```bash
# По умолчанию: 500K шагов на эксперимент
./experiments/run_all.sh

# Короткий запуск для проверки
T_MAX=100000 ./experiments/run_all.sh
```

### Запуск отдельных экспериментов

```bash
# Полный Rainbow
bash experiments/01_rainbow_pong.sh

# Или напрямую через CLI
uv run python main.py --game Pong --id rainbow_pong --t-max 500000

# Абляция: без приоритизации
uv run python main.py --game Pong --id no_priority --no-priority

# Абляция: без многошагового
uv run python main.py --game Pong --id no_multistep --multi-step 1

# Абляция: без распределения
uv run python main.py --game Pong --id no_distrib --no-distributional
```

### Построение графиков

```bash
uv run python experiments/plot_results.py
# Результат: experiments/results/learning_curves.png, experiments/results/ablation_pong.png
```

### Справка по CLI

```bash
uv run python experiments/main.py --help
```

## Результаты

Результаты сохраняются в `experiments/results/` после запуска экспериментов. Каждый каталог эксперимента содержит:
- `model.pth` — лучшая контрольная точка модели
- `metrics.pth` — метрики обучения (награды, Q-значения по точкам оценки)
